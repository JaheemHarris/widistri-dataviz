# 📊 Graph 1 : Nombre de commandes par zone
Type recommandé : `Bar Chart horizontal`

Pourquoi : Idéal pour comparer les zones facilement, surtout avec des labels longs.

✅ Schéma de référence : celui de la bar chart horizontale classique (en haut à droite du bloc Bar Charts dans l’image)

# 📈 Graph 2 : Volumes livrés par mois
Type recommandé : `Line Chart` ou `Stacked Area Chart`

Pourquoi : Utile pour voir l'évolution mensuelle et comparer plusieurs zones.

✅ Schéma de référence : line chart simple ou area chart empilé

# 🧭 Graph 3 : Commandes par type de client, zone, mois
Type recommandé : `Sunburst Chart` ou `Treemap`

Pourquoi : Pour explorer hiérarchiquement les données (TypeClient > Zone > Mois).

✅ Schéma de référence : sunburst dans la zone "Hierarchical" ou Treemap

# ⏱️ Graph 4 : Temps moyen de livraison par zone × point de vente
Type recommandé : `Grouped Bar Chart` ou `Heatmap`

Pourquoi :

Grouped bar → comparer plusieurs points de vente par zone

Heatmap → mieux si tu veux montrer les délais en densité de couleur

✅ Schéma de référence :

Grouped bar : sous le bloc Bar Charts groupés

Heatmap : dans la section dédiée (gauche centre de l’image)

# ⚠️ Graph 5 : Livraison retardée par zone + cause
Type recommandé : `Stacked Bar Chart` ou `Matrix Heatmap`

Pourquoi :

Stacked bar → voir nombre de retards et causes empilés par zone

Heatmap → plus synthétique si tu veux montrer l’intensité des retards selon les causes

✅ Schéma de référence :

Stacked Bar : en bas du bloc bar chart (barres colorées superposées)

Matrix Heatmap : blocs carrés colorés selon fréquence
